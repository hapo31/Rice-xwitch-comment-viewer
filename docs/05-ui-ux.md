# UI/UX設計

## 基本方針

配信中に横目で状態確認し、数クリックで復旧できるツールにする。派手な配信者向けダッシュボードではなく、VSCodeに近い「独自タイトルバー + 左ナビ + 作業ペイン + 下部ステータスバー」の静かなUIを採用する。

## レイアウト

```text
+----------------------------------------------------------+
| Title Bar: Rice | UI scale | minimize/maximize/close     |
+----------------------------------------------------------+
| Activity Bar | Side Panel        | Main View             |
|              |                   |                       |
| icons        | connection/queue  | chat log / login/settings |
|              | compact controls | details               |
+----------------------------------------------------------+
| Status Bar: Twitch connected | Bouyomi OK | queue 3 | ... |
+----------------------------------------------------------+
```

### Title Bar

OS標準タイトルバーは非表示にし、VSCode風の薄い独自タイトルバーを使う。ドラッグ移動、ダブルクリック最大化、最小化/最大化/閉じる、四辺/四隅リサイズはTauriのウィンドウAPIへ委譲する。

タイトルバーにはUI倍率の切替を置く。初期値は自動で、必要に応じて 100% / 125% / 150% を選べる。

### Activity Bar

幅48px程度の縦ナビゲーション。アイコン中心で、hover tooltipを出す。

- Chat
- Launcher
- Queue
- Filter
- Settings
- Login
- Logs

アイコンは既存フロントエンド構成に合わせ、なければ `lucide-react` を採用する。

### Side Panel

現在ビューに応じた補助ペイン。

Chat:

- Twitch接続状態
- 接続チャンネル
- 読み上げON/OFF
- 最近の警告

Queue:

- 一時停止/再開
- スキップ
- クリア
- キュー件数

Login:

- Twitch認証
- 接続チャンネル
- 起動時自動接続

Settings:

- 棒読みちゃん接続先
- 棒読みちゃんの速度/音程/音量/声質
- 読み上げ基本設定
- emote 読み上げ
- テスト読み上げ

### Main View

主要な作業領域。

初期画面はChat viewにする。

- 上部: 接続中チャンネル、読み上げ状態、テスト読み上げボタン
- 中央: チャットログ
- 右寄せまたは下部: 選択チャット詳細

## 画面一覧

| 画面 | 内容 |
| --- | --- |
| Chat | Twitchチャットのライブログ。読まれた/スキップ/NG適用を視覚的に区別する。 |
| Launcher | 配信でよく使うWindowsアプリを正方形タイルで一覧表示し、登録、削除、単体起動、一斉起動を行う。作業領域を広く使うため、この画面ではSide Panelを覆って表示する。 |
| Queue | 読み上げ待ち、読み上げエラー、フィルター設定で読み飛ばしたチャットの一覧。Chat と同じく新しい項目を上に表示し、手動スキップ、再読込、削除ができる。読み上げ済みと手動スキップ済みは表示しない。 |
| Filter | 読み上げるチャットの種類、NGユーザー、NGワード、URL処理、長文処理。 |
| Settings | 起動時接続、棒読みちゃん接続先、速度/音程/音量/声質、名前読み上げ、emote 読み上げ、テスト読み上げ。 |
| Login | Twitch認証、接続チャンネル。読み上げ設定は置かない。 |
| Logs | EventSub、認証、読み上げアダプタのログ。 |

## Launcherタイル

- Windows 10のスタートメニューに近い正方形タイルとし、背景色、中央のアプリアイコン、下端のアプリ名で構成する。
- アプリ名は1行で省略表示し、タイル右下の三点リーダーから「削除」を開く。
- 一覧末尾に破線の追加用スケルトンタイルを常設し、`.exe` / `.lnk` の複数選択に対応する。
- 画面全体でファイルのドラッグ＆ドロップを受け付け、ドロップ中だけ控えめなオーバーレイを表示する。
- ヘッダー右側に「一斉に起動」を置き、下端の細い案内領域へ操作説明と直近結果を表示する。
- `order`, `backgroundColor`, `groupId`, `kind` を表示モデルの境界に残し、後続で並べ替え、色変更、枠付きグループ、Webリンクを追加できるようにする。

## Tailwindデザイン指針

色:

- 背景: `zinc-950`, `zinc-900`
- パネル: `zinc-900`, `zinc-850`相当のカスタム
- 境界線: `zinc-800`
- テキスト: `zinc-100`, `zinc-400`
- アクセント: `sky-400` と `emerald-400`
- エラー: `rose-400`
- 警告: `amber-400`

コンポーネント:

- カード乱用はしない。パネルと行ベースで構成する。
- ボタンはアイコンボタン中心。ラベルが必要な主要操作だけテキスト付きにする。
- 接続状態はステータスバーと小さなドットで表示する。
- チャット行は高さを安定させ、長文は折りたたみまたは2行までにする。
- 配信中に誤操作しやすい「クリア」「切断」は確認を挟む。

高DPI対応:

- ウィンドウを異なるDPIのモニタへ移動しても横目で読めるよう、`devicePixelRatio` の変化に応じて基準フォントサイズを自動調整する。
- 調整は `rem` ベースの文字と余白を中心に効かせ、VSCode風の密度感を大きく崩さない範囲に留める。
- WSLgなど自動検出が不安定な環境に備え、タイトルバーから手動倍率へ切り替えられるようにする。

## チャットログ行

表示項目:

- 時刻
- ユーザー表示名
- バッジ簡易表示
- チャット本文
- 読み上げ状態アイコン

状態:

- `queued`: 読み上げ待ち
- `spoken`: 読み上げ済み
- `skipped`: スキップ
- `blocked`: ルールで除外
- `error`: 読み上げ失敗

## キーボード操作

- `Space`: 読み上げ一時停止/再開
- `S`: 現在の読み上げをスキップ
- `Cmd/Ctrl+K`: コマンドパレット
- `Cmd/Ctrl+,`: 設定

コマンドパレットはMVP後でよいが、VSCode風UIとの相性がよい。
