# UI/UX設計

## 基本方針

配信中に横目で状態確認し、数クリックで復旧できるツールにする。派手な配信者向けダッシュボードではなく、VSCodeに近い「独自タイトルバー + 左ナビ + 作業ペイン + 下部ステータスバー」の静かなUIを採用する。

## レイアウト

```text
+----------------------------------------------------------+
| Title Bar: Rice | UI scale | minimize/maximize/close     |
+----------------------------------------------------------+
| Activity Bar | Side Panel        | Main View             |
|              |                   |                       |
| icons        | connection/queue  | chat log / settings   |
|              | compact controls | details               |
+----------------------------------------------------------+
| Status Bar: Twitch connected | Bouyomi OK | queue 3 | ... |
+----------------------------------------------------------+
```

### Title Bar

OS標準タイトルバーは非表示にし、VSCode風の薄い独自タイトルバーを使う。ドラッグ移動、ダブルクリック最大化、最小化/最大化/閉じる、四辺/四隅リサイズはTauriのウィンドウAPIへ委譲する。

タイトルバーにはUI倍率の切替を置く。初期値は自動で、必要に応じて 100% / 125% / 150% を選べる。

### Activity Bar

幅48px程度の縦ナビゲーション。アイコン中心で、hover tooltipを出す。

- Chat
- Queue
- Rules
- Voices
- Settings
- Logs

アイコンは既存フロントエンド構成に合わせ、なければ `lucide-react` を採用する。

### Side Panel

現在ビューに応じた補助ペイン。

Chat:

- Twitch接続状態
- 接続チャンネル
- 読み上げON/OFF
- 最近の警告

Queue:

- 一時停止/再開
- スキップ
- クリア
- キュー件数

Settings:

- Twitch認証
- 棒読みちゃん接続先
- 読み上げフォーマット

### Main View

主要な作業領域。

初期画面はChat viewにする。

- 上部: 接続中チャンネル、読み上げ状態、テスト発話ボタン
- 中央: コメントログ
- 右寄せまたは下部: 選択コメント詳細

## 画面一覧

| 画面 | 内容 |
| --- | --- |
| Chat | Twitchコメントのライブログ。読まれた/スキップ/NG適用を視覚的に区別する。 |
| Queue | 読み上げ待ち一覧。手動スキップ、再読込、削除ができる。 |
| Rules | NGユーザー、NGワード、URL処理、長文処理、emote処理。 |
| Voices | 棒読みちゃんの速度/音程/音量/声質、テスト発話。 |
| Settings | Twitch認証、接続チャンネル、保存設定、起動時自動接続。 |
| Logs | EventSub、認証、読み上げアダプタのログ。 |

## Tailwindデザイン指針

色:

- 背景: `zinc-950`, `zinc-900`
- パネル: `zinc-900`, `zinc-850`相当のカスタム
- 境界線: `zinc-800`
- テキスト: `zinc-100`, `zinc-400`
- アクセント: `sky-400` と `emerald-400`
- エラー: `rose-400`
- 警告: `amber-400`

コンポーネント:

- カード乱用はしない。パネルと行ベースで構成する。
- ボタンはアイコンボタン中心。ラベルが必要な主要操作だけテキスト付きにする。
- 接続状態はステータスバーと小さなドットで表示する。
- コメント行は高さを安定させ、長文は折りたたみまたは2行までにする。
- 配信中に誤操作しやすい「クリア」「切断」は確認を挟む。

高DPI対応:

- ウィンドウを異なるDPIのモニタへ移動しても横目で読めるよう、`devicePixelRatio` の変化に応じて基準フォントサイズを自動調整する。
- 調整は `rem` ベースの文字と余白を中心に効かせ、VSCode風の密度感を大きく崩さない範囲に留める。
- WSLgなど自動検出が不安定な環境に備え、タイトルバーから手動倍率へ切り替えられるようにする。

## コメントログ行

表示項目:

- 時刻
- ユーザー表示名
- バッジ簡易表示
- コメント本文
- 読み上げ状態アイコン

状態:

- `queued`: 読み上げ待ち
- `spoken`: 読み上げ済み
- `skipped`: スキップ
- `blocked`: ルールで除外
- `error`: 読み上げ失敗

## キーボード操作

- `Space`: 読み上げ一時停止/再開
- `S`: 現在発話をスキップ
- `Cmd/Ctrl+K`: コマンドパレット
- `Cmd/Ctrl+,`: 設定

コマンドパレットはMVP後でよいが、VSCode風UIとの相性がよい。
